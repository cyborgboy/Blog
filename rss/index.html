Introduction :

Web-storage API in HTML 5 includes both Local Storage and Session Storage where Local Storage is mainly built to store information in browser via a JavaScript code . Unlike Cookies and Session storage the information stored in Local storage can hold up Larger data  for longer time and a separate  individual portions are made for different users sessions on same browser.

Sample JavaScript code on server side application  that instructs the browsers  to store information in Local storage :

Local storage improves the performance of web Application by storing web states and setting information at client side which  helps in  creating  more lightweight applications so it is been quite used by developers for storing information as they found it cooler then cookie and session storage but using the local storage for storing sensitive information can affect large number of users

So How exactly Local Storage is vulnerable to different type of Web Vulnerabilities ?

Cross site Scripting ( XSS ) :

We know there are lot's of Cross site Scripting vulnerabilities ( XSS) discovered daily and which stands top 10 in web  vulnerabilities ranking  , so if a local storage is exposed to a website vulnerable to cross site scripting ( xss ) attack  then all your data ( sessions and password )  will be stolen and it is no more secure.

Exploiting the Local Storage to steal information via XSS :

Like for retrieving cookies we use <script>alert(document.cookie)</script> so calling <script>alert(localstorage)</script> wont work in this scenario , as Local storage stores data in array format we need a specific key to call the data from that array structure  localStorage.getitem('key')  it requires brute forcing to match exact key so here we use JSON.stringify(localstorage);
